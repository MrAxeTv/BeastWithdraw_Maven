# Project Notes For Codex

## Feature Tracking

- Keep [`FEATURE_TRACKER.yml`](./FEATURE_TRACKER.yml) updated whenever a user-facing feature is added or changed.
- Update that file in the same task whenever you add:
  - commands
  - permissions
  - config options
  - major behavior changes
  - logging or storage paths
- Prefer short YAML entries with exact command names, permission nodes, and config paths.
- Treat `FEATURE_TRACKER.yml` as the source file for later wiki preparation.
